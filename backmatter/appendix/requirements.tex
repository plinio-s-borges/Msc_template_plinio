\chapter{Requirements Appendix}\label{app:requirements}

This appendix provides an example of how to use the requirements tracking system. 
You can define requirements using the `.\\requirement` command or creating an inline requirement with `.\\inlineReq{R-ID}` and reference them in your text using `.\\reqRef{R-ID}` .

The `.\\requirement` command has the following structure:
% \requirement{<ID>}{<Title>}{<Description>}{<Type>}{<Tags>}
% - <ID>: A unique identifier for the requirement (e.g., R-01).
% - <Title>: A short, descriptive title.
% - <Description>: A detailed explanation of the requirement.
% - <Type>: The category of the requirement (e.g., Functional, Non-Functional, Performance, Security).
% - <Tags>: Comma-separated tags for filtering or grouping (e.g., UI, Database, API).

\section{Example System Requirements}

% Example of a Non-Functional, Performance requirement tagged with 'UI'
\requirement{R-01}{System Performance}{The system shall respond to user input within 2 seconds.}{Performance}{UI}

% Example of a Functional requirement tagged with 'Database'
\requirement{R-02}{Data Storage}{The system shall store up to 1 TB of user data.}{Functional}{Database}

\section{Example User Requirements}

% Example of a Functional requirement related to Security
\requirement{R-03}{User Authentication}{Users shall be able to log in using a username and password.}{Functional}{Security}

% Example of a Functional requirement for the UI
\requirement{R-04}{Export Data}{Users shall be able to export their data to a CSV file.}{Functional}{UI}

\section{Referencing Requirements}

Once requirements are defined, you can reference them from anywhere in your document. This is useful for tracing how the system's design and implementation satisfy the initial requirements.

\begin{itemize}
    \item The \textbf{inline requirement} command, `\inlineReq{R-ID}`, is used when a requirement is first introduced or satisfied in the text. For example, the implementation of the user login feature satisfies requirement \inlineReq{R-03}. This creates a back-link from the requirement definition to this point in the text.
    \item The standard \textbf{requirement reference} command, `\reqRef{R-ID}`, is used for subsequent mentions. For instance, the performance of the system is critical, as noted in \reqRef{R-01}, and the data storage capacity is defined by \reqRef{R-02}. The ability for users to export their data is covered in \reqRef{R-04}.
\end{itemize}
